# Refined Plan: User DTO Implementation

This plan outlines the steps to implement the `UserDTO` classes in `src/user/user.dto.ts`. These DTOs will facilitate data validation and Swagger documentation for User-related API endpoints, leveraging existing Zod schemas.

## Objectives
- Create DTO classes for User creation, updates, and queries using `nestjs-zod`'s `createZodDto`.
- Implement Swagger-compatible response DTOs using standard response wrappers.

## Prerequisites
- [x] Prisma schema defined for `User`.
- [x] Zod schemas defined in `src/user/user.schema.ts` (`UserCreateSchema`, `UserPresentationSchema`, `UserQueryParamsSchema`, `UserListSchema`).

## 1. DTO Implementation
Implement the `UserDTO` classes in `src/user/user.dto.ts`.

- [x] **Import Dependencies:**
  - `createZodDto` from `nestjs-zod`.
  - Import the following schemas from `./user.schema.ts`:
    - `UserCreateSchema`
    - `UserPresentationSchema`
    - `UserQueryParamsSchema`
    - `UserListSchema`
  - Import response schema helpers from `@src/_core/schemas/response-success.schema`:
    - `makeSuccessResponseSchema`
    - `makePaginatedResponseSchema`
- [x] **Define Main DTO Classes:**
  - `UserCreateDTO`: Use `createZodDto(UserCreateSchema)`.
  - `UserQueryParamsDTO`: Use `createZodDto(UserQueryParamsSchema)`.
  - `UserPresentationDTO`: Use `createZodDto(UserPresentationSchema)`.
- [x] **Define Swagger Response Wrappers:**
  - `UserResponseDTO`: Use `createZodDto(makeSuccessResponseSchema(UserPresentationSchema))`.
  - `UserListResponseDTO`: Use `createZodDto(makePaginatedResponseSchema(UserListSchema))`.

## 2. Validation Steps
Ensure the newly created DTOs are correctly defined and do not introduce errors.

  - Run type checking and build:
