# User Get One Detail Plan

Implementation of `GET /users/:id` endpoint with Role-Based Access Control (RBAC).

## Context
- Module: `src/user`
- Existing files: `user.dto.ts`, `user.repository.ts`, `user.module.ts`, `user.schema.ts`

## Requirements
- Admin can retrieve any user's detail.
- Non-admin (regular user) can only retrieve their own detail.
- Accessing other user's detail by non-admin should return `403 Forbidden`.
- Swagger documentation for the new endpoint.

## Proposed Implementation

### 1. Service Layer
- [x] Create `src/user/user.service.ts`
  - [x] Implement `findOne(id: string, currentUser: { id: string, role: string })`
  - [x] Add logic: If `currentUser.role !== 'admin'` AND `id !== currentUser.id`, throw `ForbiddenException`.
  - [x] Call `UserRepository.findById(id)`.
  - [x] Throw `NotFoundException` if user doesn't exist.
  - [x] Return the user object (mapped to presentation schema in controller).

### 2. Controller Layer
- [x] Create `src/user/user.controller.ts`
  - [x] Add `@ApiTags('users')`, `@ApiBearerAuth()`, `@UseGuards(JwtAuthGuard)`.
  - [x] Inject `UserService`.
  - [x] Implement `GET /users/:id` endpoint:
    - [x] `@Get(':id')`
    - [x] Use `@Param('id') id: string`
    - [x] Use `@Req() req: AuthenticatedRequest` to get `currentUser`.
    - [x] `@ApiResponse` with `UserResponseDTO`.
    - [x] `@ApiErrors401()` and other relevant error decorators.

### 3. Module Registration
- [x] Update `src/user/user.module.ts`
  - [x] Add `UserService` to `providers`.
  - [x] Add `UserController` to `controllers`.
