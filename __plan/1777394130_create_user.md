# Create User Plan

Implement create user flow in the User module, allowing administrators to manually create new users with hashed passwords.

## Objectives
- Update User schema to include password for creation.
- Implement create user logic in `UserService` using `argon2` for hashing.
- Expose `POST /users` endpoint in `UserController` restricted to admin users.
- Ensure proper Swagger documentation for the new endpoint.
- Verify implementation with unit and E2E tests.

## Prerequisites
- `argon2` is already installed and used in the `Auth` module.
- `UserRepository` already has a `create` method.

## Implementation Steps

### 1. Schema & DTO Updates
- [x] Update `src/user/user.schema.ts`:
    - Modify `UserCreateSchema` to include `password` (e.g., `z.string().min(8)`).
    - Ensure `role` is also included if admins should be able to set it.
- [x] Verify `src/user/user.dto.ts` uses the updated `UserCreateSchema`.

### 2. Service Implementation
- [x] Update `src/user/user.service.ts`:
    - [x] Import `argon2`.
    - [x] Implement `create(dto: UserCreateDTO)`:
        - [x] Validate DTO using `UserCreateSchema.parse()`.
        - [x] Check for existing user with same email or username (optional but recommended for better error messages).
        - [x] Hash the password using `argon2.hash()`.
        - [x] Call `this.userRepository.create()` with the hashed password.
        - [x] Return the created user parsed with `UserPresentationSchema`.

### 3. Controller Implementation
